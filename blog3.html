Excursion: feature selection in high dimensional data

I paused the fast.ai course for a while as I am back to my day to day work as a bioinformatician. But I will resume this work as part of an upcoming project, that tightly relates to this blog entry.

Lately, I have been thinking a lot about feature selection. In the sphere of genetic/epigenetic or other Omics data, we are dealing with well over half a million of features. That sounds like a deeply phenotyped data set with lots of detailed information on the individual instances. But on top of this high dimensional feature space, we are also frequently dealing with a very low number of instances. Not dealing with image data, we want to make sure that we are selecting a meaningful set of features for our model. This is especially important to prevent overfitting.

Generally, there are three different feature selection approaches:

  1. Filtering
  2. Wrapper Methods
  3. Embedded Methods
The filtering approach performs feature selection as a seperate step to the modelling. Before a model is trained, important features can be extracted by an algorithm. The second approach, wrapper, trains a model on a subset of features, evaluates the model, then retrains the model with a different set of features, selecting the model and features with the best performance as the final model. Lastly, the embedded method is computationally more efficient than the two previously mentioned methods. In this approach, feature selection and classifier/score are performed at the same time. An example for this would be lasso or elastic net regression. Lasso and elastic net are both part of the so called penalization class of feature selection models.

Apart from feature selection approaches, there are also so called dimension reduction methods. Priciple Component Analysis or tSNE are exampels for this. Those methods are extremely helpful when we want to visualise a high dimensional space.

Figure 1.: An example of a principle component plot, using 71 metabolite features. Rauschert et al.

But the features itself are very hard if not at all interpretable anymore with the dimension reduction. When it comes to festure selection, I would probably suggest using either elastic net regression or random forest. Both are embedded methods and when one has the large enough memory available, they are probably most efficient.

That's it for this time.
